# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```sh
# Run all tests
npm test

# Run a single test file
npx jest tests/text.test.js

# Run a single test by name
npx jest -t "add text to a label"

# Build the browser bundle (outputs to dist/jszpl.bundle.js)
npx webpack

# Repack font data
npm run font-packer
```

## Architecture

JSZPL is a Node.js library (also bundled for browser use) that generates ZPL II printer commands from a JavaScript component tree.

**Entry point**: `src/jszpl.js` — re-exports all public classes and enums.

**Component hierarchy** (`src/components/`):
- `BaseComponent` — minimal base; defines `generateZPL`, `generateXML`, `generateBinaryImage` methods
- `BaseVisualComponent extends BaseComponent` — adds layout properties: `width`, `height`, `left`, `top`, `margin`, `fixed`, `invert`, `grid`
- `BaseGraphicComponent extends BaseVisualComponent` — adds `border`, `fill`, `padding`, `content` (child array); used by container elements
- `BaseContainerComponent extends BaseGraphicComponent` — used by Grid specifically
- Leaf components: `Text`, `Barcode`, `SerialNumber`, `Line`, `Raw`
- Container components: `Label`, `Box`, `Circle`, `Grid`, `Graphic`

**Properties** (`src/properties/`): Typed wrapper objects (`Size`, `Spacing`, `Alignment`, `FontFamily`, `PrintDensity`, `GraphicData`, `BarcodeType`, `GridPosition`). Most accept enum values from `src/enums/`.

**Rendering**: Each component implements `generateZPL(offsetLeft, offsetTop, availableWidth, availableHeight, widthUnits, heightUnits)`. The Label component drives the render tree — calling `generateZPL()` on `Label` recursively generates ZPL for all children. `generateBinaryImage()` produces a pixel grid (array of arrays) for preview rendering.

**Helpers** (`src/helpers/`):
- `label-tools.js` — coordinate and size calculation utilities
- `image-processor.js` — converts RGBA pixel data to 1-bit arrays for `Graphic`
- `image-resizer.js` — scales image data to fit element bounds
- `zpl-image-tools.js` — encodes binary pixel arrays into ZPL `^GF` graphic field format
- `barcode-renderer.js` — renders barcode pixel data for `generateBinaryImage`

**Fonts**: Embedded as base64 in `src/b64-fonts.js` (generated by `font-packer/font-packer.js`). Fonts A–F are supported; G–V are not implemented.

**Tests**: Jest-based in `tests/`. Tests import from `src/jszpl.js` directly and use `tests/test-helpers.js` for common label setup. Many tests assert exact ZPL string output and pixel counts from `generateBinaryImage`.

**Browser bundle**: `webpack.config.js` bundles `src/jszpl.js` as a UMD module to `dist/jszpl.bundle.js`.
